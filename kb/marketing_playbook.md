# Marketing Playbook (Internal Only)

**IMPORTANT**: This playbook is for MARKETING mode only. Never use this content in OPERATOR mode customer interactions.

## Content Strategy

### Messaging Pillars
1. Proven workflows over customization
2. Standardized success
3. Manufacturing-focused solutions
4. Network effects (ShopLink)

### Tone Guidelines
- Professional but approachable
- Confident but not overpromising
- Focus on outcomes, not features
- Avoid technical jargon when possible

## Content Types

### Blog Posts
- How-to guides
- Case studies (with permission)
- Industry insights
- Product updates (factual only)

### Email Campaigns
- Onboarding sequences
- Feature announcements
- Best practices
- Re-engagement campaigns

### Landing Pages
- Product pages
- Feature pages
- Pricing pages
- Resource pages

### Ad Copy
- Search ads
- Social media ads
- Retargeting campaigns

## Claims Verification

**CRITICAL**: All factual claims must be verified against:
- `config/pricing.json` for pricing
- `kb/product_overview.md` for features
- `kb/faq.md` for policies

Flag any unverified claims in claims checklist.

## Hypothetical Positioning

When proposing positioning ideas:
- Label as "hypothesis" or "proposed positioning"
- Never present as commitments
- Include reasoning
- Suggest testing approach

## Content Variants

When creating content drafts:
- **Short**: Concise, punchy, attention-grabbing
- **Medium**: Balanced, informative, engaging
- **Aggressive**: Bold claims, strong CTAs, urgency

## Objection Handling

Common objections and responses:
- "Too expensive" → Value proposition, ROI calculator
- "Need customization" → Standardized success stories
- "Not sure if it fits" → Qualification questions, trial
- "Competitor has X" → Our unique value, network effects

## Campaign Planning

When creating campaign plans:
- Define target audience
- Set clear objectives
- Choose channels
- Define success metrics
- Include budget estimates (if applicable)
- Timeline (hypothetical, not commitment)
